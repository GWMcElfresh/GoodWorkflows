# GoodWorkflows on Bazzite

Quickstart for running GoodWorkflows DSL2 Nextflow pipelines on a Bazzite (Fedora-based) workstation with an NVIDIA GPU.

## Prerequisites

- **Bazzite** (or any Fedora-based Linux with Podman)
- **NVIDIA GPU** (RTX 3070 or similar, 8+ GB VRAM)
- **Podman** (pre-installed on Bazzite)
- **System packages**: `java-25-openjdk`, `libcurl-devel`, `libuv`, `cmake`, `openssl-devel`, `libxml2-devel` (checked by `setup.sh`)
- **R** (for `fetch_example_data.sh` — install with `rpm-ostree install R`)
- **R packages**: `Seurat`, `SeuratData`, `babelgene` (auto-checked by `fetch_example_data.sh`)

## Quickstart

```bash
# 1. Clone the repo
git clone https://github.com/GWMcElfresh/GoodWorkflows.git
cd GoodWorkflows/template/gw

# 2. One-time setup (Nextflow, Podman, GPU verify, container pull)
bash setup.sh

# 3. Download example data (pbmc3k split into 3 pseudo-species)
bash fetch_example_data.sh

# 4. Run a workflow
bash run.sh --workflow ingest_export     # CPU-only: download + export counts
bash run.sh --workflow integration       # Full GPU pipeline: ingest → export → harmonize → scMODAL
```

## What Gets Created

```
template/gw/
├── setup.sh                  # Bootstrap script
├── run.sh                    # Workflow launcher
├── fetch_example_data.sh     # Download + split pbmc3k
├── README.md                 # This file
├── samplesheet.csv           # Generated by fetch_example_data.sh
├── data/                     # Downloaded RDS files
│   ├── pbmc3k_human.rds
│   ├── pbmc3k_macaque.rds
│   └── pbmc3k_mouse.rds
└── runs/                     # Per-run output directories
    └── <workflow>_<timestamp>/
        ├── outputs/          # Pipeline results
        ├── work/             # Nextflow work directory
        └── logs/             # Nextflow logs
```

## Profiles

| Profile | Description |
|---|---|
| `local_gpu` | **Default.** Podman + NVIDIA GPU passthrough (`--gpus all --privileged`), maxForks=1 |
| `local` | Podman, CPU-only, blocks GPU workflows |
| `test` | CI smoke tests with stub blocks |

## Workflows

| Workflow | GPU Required | Description |
|---|---|---|
| `ingest_export` | No | Download Seurat RDS + export 10x-like counts |
| `ingest_tabulate` | No | Download metadata + build subjectIdTable |
| `integration` | **Yes** | Full pipeline: ingest → export → harmonize → scMODAL |

## Samplesheet Format

Three modes are supported:

### Local File Mode (data already on disk)
```csv
sample_id,output_file_id,url,path,species
PBMC_HUMAN,,,,/path/to/pbmc3k_human.rds,human
PBMC_MACAQUE,,,,/path/to/pbmc3k_macaque.rds,macaque
```

### URL Mode (public data download)
```csv
sample_id,output_file_id,url,path,species
PBMC_HUMAN,,https://example.com/pbmc3k_human.rds,,human
PBMC_MACAQUE,,https://example.com/pbmc3k_macaque.rds,,macaque
```

### LabKey Mode (requires .netrc credentials)
```csv
sample_id,output_file_id,url,path,species
SAMPLE_01,100001,,,human
SAMPLE_02,100002,,,macaque
```

The INGEST dispatcher auto-detects which mode to use based on which column (`path`, `url`, or `output_file_id`) is non-empty. Exactly one must be present per row.

## Custom Runs

```bash
# Use your own samplesheet
bash run.sh --workflow integration --input /path/to/my/samplesheet.csv

# Pass extra Nextflow params
bash run.sh --workflow integration --species_order human,macaque,mouse --scmodal_latent 30

# Run CPU-only workflows
bash run.sh --workflow ingest_export
bash run.sh --workflow ingest_tabulate --tabulate_id_cols cDNA_ID,SubjectId
```

## Troubleshooting

### GPU passthrough fails
```bash
# Verify nvidia-smi works on the host
nvidia-smi

# Check CDI config
ls /etc/cdi/nvidia.yaml

# If missing, install nvidia-container-toolkit
sudo rpm-ostree install nvidia-container-toolkit
```

### "Samplesheet row must have one of 'output_file_id', 'url', or 'path'"
Your samplesheet must have exactly one of `output_file_id` (LabKey), `url` (download), or `path` (local file) non-empty per row.

### Container pull fails
Ensure the images are public:
- `ghcr.io/bimberlabinternal/rdiscvr:latest`
- `ghcr.io/bimberlabinternal/cellmembrane:latest`
- `ghcr.io/gwmcelfresh/scmodal:latest`

### R packages not found
```bash
# Install Seurat, SeuratData, and babelgene
R -e 'install.packages(c("Seurat"), repos = "https://cloud.r-project.org")'
R -e 'remotes::install_github("satijalab/seurat-data")'
R -e 'install.packages("babelgene", repos = "https://cloud.r-project.org")'
